Data Flow
1

User Selects Filter

FilterBar component updates reactive state in useFilters() composable — Time Period, Warehouse, Category, or Order Status.

2

View Component Reacts

Page views watch filter state and call api.js methods with the current filter object. Values of 'all' are stripped from the URL params.

3

HTTP Request to FastAPI

Axios sends a GET request to localhost:8001/api/... with query parameters: ?warehouse=Tokyo&category=Sensors&month=2025-03

4

Backend Filters In-Memory

apply_filters() runs on the in-memory list. Warehouse uses exact match; category and status use case-insensitive match. Month supports both YYYY-MM and quarter strings like Q1-2025.

5

Pydantic Validates & Serializes

Response data is validated against Pydantic models before serialization to JSON. HTTPException(404) is raised for missing resources.

6

Vue Renders Updated View

Component reactive refs are updated with the response. Computed properties derive aggregates (totals, percentages). Tables and custom SVG charts re-render.

Design Constraints & Known Limits
  • ⚠
    No persistence — All data resets on server restart. JSON files are read-only on boot; only purchase orders are written to memory at runtime.
  • ⚠
    No pagination — All records are returned in a single response. Suitable for the current dataset size (~dozens of items per entity).
  • ⚠
    No authentication — useAuth.js has a hardcoded mock user. isAuthenticated always returns true. CORS is open to all origins.
  • ⚠
    Inventory has no time dimension — The month filter does not apply to /api/inventory. Only warehouse and category filters are supported.
  • ⓘ
    Revenue goals — Dashboard targets are hardcoded: $800K/month (single month) and $9.6M YTD (all months view).
  • ⓘ
    Direct API URL — Frontend calls localhost:8001 directly. No Vite proxy is configured, so both servers must be running simultaneously.
